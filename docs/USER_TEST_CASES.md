# 🧾 User Test Cases — Daily Briefing Agent

사람이 직접 실행하며 검증하는 **수동 인수 테스트(UAT, User Acceptance Test)** 입니다.
자동 단위 테스트(`pytest`)가 다루지 못하는 **실제 API 실행 경로**를 사람이 확인합니다.

## 사용법
- 각 케이스를 위에서 아래로 순서대로 실행하세요.
- `결과` 칸에 ✅(통과) / ❌(실패)를 기록하세요.
- ❌가 나오면 `비고`에 증상을 적고, 해당 단계의 디버깅 팁을 참고하세요.

## 자동 vs 수동 테스트 역할 분담

| | 자동 (pytest) | 수동 (이 문서) |
|---|---|---|
| 검증 대상 | 내부 로직 (계약·재시도·파싱) | 실제 API 실행 경로·출력 품질 |
| API 호출 | 없음 (모킹) | 있음 (실제 키) |
| 실행 주체 | CI/개발자 | 사람 (눈으로 품질 확인) |
| 속도 | 빠름 (2초) | 느림 (분 단위) |
